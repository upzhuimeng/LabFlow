LabFlow 安装指南
================================================================================

一、环境要求
--------------------------------------------------------------------------------

Python: >= 3.11
Node.js: >= 18.0
MySQL: >= 8.0
uv: >= 0.5（Python 包管理器，推荐使用）
npm: >= 9.0

--------------------------------------------------------------------------------

二、数据库初始化
--------------------------------------------------------------------------------

2.1 创建数据库

方式一：使用 SQL 脚本（推荐）
执行命令：mysql -u root -p < docs/DATABASE_SCHEMA.sql

方式二：手动执行
CREATE DATABASE IF NOT EXISTS labflow DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE labflow;
然后执行 docs/DATABASE_SCHEMA.sql 中的建表语句

--------------------------------------------------------------------------------

2.2 导入测试数据（可选）

执行命令：mysql -u root -p labflow < docs/TEST_DATA.sql

三、后端初始化
--------------------------------------------------------------------------------

方式一：使用 uv（推荐）

1. 进入后端目录
   cd backend

2. 安装依赖
   uv sync

3. 配置环境变量
   cp .env.example .env
   编辑 .env，修改 DB_PASSWORD 和 JWT_SECRET_KEY

4. 启动服务
   uv run -m app.main

--------------------------------------------------------------------------------

方式二：传统 Python 虚拟环境

1. 进入后端目录
   cd backend

2. 创建虚拟环境
   python3 -m venv .venv

3. 激活虚拟环境
   Linux/macOS：source .venv/bin/activate
   Windows：.venv\Scripts\activate

4. 安装依赖
   pip install -r requirements.txt

5. 配置环境变量
   cp .env.example .env
   编辑 .env

6. 启动服务
   python -m app.main

--------------------------------------------------------------------------------

四、前端初始化
--------------------------------------------------------------------------------

4.1 安装依赖

进入前端目录：cd frontend
执行命令：npm install

--------------------------------------------------------------------------------

4.2 配置环境变量

方式一：cp .env.local.example .env.local
方式二：手动创建 .env.local

编辑 .env.local，设置 API 地址：
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000/api/v1

说明：
开发环境可使用 http://localhost:8000/api/v1
生产环境使用 https://your-domain.com/api/v1

--------------------------------------------------------------------------------

4.3 构建和运行

开发模式：npm run dev

生产构建：npm run build
生产运行：npm run start

--------------------------------------------------------------------------------

五、Nginx 部署配置（可选）
--------------------------------------------------------------------------------

参考 scripts/nginx/ 目录下的配置文件

开发环境：
1. sudo cp scripts/nginx/dev.conf /etc/nginx/sites-available/labflow
2. sudo ln -s /etc/nginx/sites-available/labflow /etc/nginx/sites-enabled/
3. sudo nginx -t && sudo systemctl reload nginx

生产环境（需配置 HTTPS）：
1. sudo cp scripts/nginx/prod.conf /etc/nginx/sites-available/labflow
2. 编辑 prod.conf 修改域名和证书路径
3. sudo ln -s /etc/nginx/sites-available/labflow /etc/nginx/sites-enabled/
4. sudo nginx -t && sudo systemctl reload nginx

--------------------------------------------------------------------------------

六、重要配置说明
--------------------------------------------------------------------------------

6.1 环境变量

后端 .env 关键配置：

DB_PASSWORD：数据库密码（必须修改）
JWT_SECRET_KEY：JWT 密钥（生产环境必须使用强随机密钥）
AGENT_API_KEY：AI API Key（智能助手功能必需）

生成强随机 JWT 密钥：
openssl rand -hex 64

--------------------------------------------------------------------------------

6.2 HTTPS 要求

重要：生产环境必须使用 HTTPS！

系统前后端通过 HTTP 明文传输数据（包括 JWT Token），存在安全风险。

开发环境：可使用自签证书 + 浏览器手动信任
生产环境：必须使用 Let's Encrypt 或其他可信 CA 签发的证书

--------------------------------------------------------------------------------

6.3 AI 功能配置

智能助手支持 OpenAI 兼容 API（如 MiniMax、SiliconFlow、Iflow 等，大部分模型厂商均支持）

AGENT_MODEL_NAME：模型名称
AGENT_API：API 地址
AGENT_API_KEY：API Key

八、常见问题
--------------------------------------------------------------------------------

问：数据库连接失败？

答：检查 .env 中的 DB_PASSWORD 是否正确，确保 MySQL 服务正在运行。

--------------------------------------------------------------------------------

问：AI 功能效果不佳（如返回文本而非结构化数据、推荐结果不准确）？

答：模型能力对该功能影响较大，建议更换为工具调用能力更强或经过强化训练的模型，
如 Qwen3.5/Qwen3.6、GPT-4o 等支持 function calling 的模型。
本项目测试使用的模型为 MiniMax-M2.7。

可选的 API 方案：
- SiliconFlow（硅基流动，知名的模型聚合平台，其中 Qwen3.5-4B 等小参数模型可免费调用，
  但不建议调用发布时间过早的模型，这类模型往往工具调用能力略差）
- Iflow（心流开放平台，其中 Qwen3、Kimi2 等大参数模型支持免费使用，
  但请注意：该平台将于 2026 年 4 月 17 日关闭服务）
- LM Studio/Ollama（本地部署，通过图形化界面本地部署模型，需要一定硬件资源，
  但高度保证了信息安全）
- 其他支持 OpenAI API 格式的平台

--------------------------------------------------------------------------------

问：前端无法登录？

答：检查后端是否正常运行（http://localhost:8000），以及 .env.local 中的
NEXT_PUBLIC_API_BASE_URL 是否正确。

--------------------------------------------------------------------------------

问：端口被占用？

答：查看端口占用：lsof -i :8000（后端）或 lsof -i :3000（前端）
    或修改 .env 中的 PORT
